Personal Data Privacy Ordiance (since 1996)

  • DPP1 - Collection

  • collection must be related to a legitimate business purpose
  • collected in a fair and lawful manner
  • amount collected must be just adequate for purpose and not excessive
  • must explicitly inform person on or before collection

  • DPP2 - Accuracy and Retention

  • personal data must be kept accurate relating to the purpose
  • personal data must not be kept longer than is necessary for the purpose it was collected for

  • DPP3 - Use

  • an individual's explicit consent must be obtained if using personal data for purposes not stated at time of collection
  • use includes disclosure to other parties and using data processors
  • a data processor is defined as any third party that processes personal data on behalf, does not use it for its own purposes
  • do not retain personal data longer than is necessary
  • have adequate security to protect personal data

  • DPP4 - Security

  • documents containing personal data should be classified as "CONFIDENTIAL"
  • protect personal data by following the control measures detailed in the information classification & handling standard

  • DPP5 - Information to be available

  • make privacy policy statement readily available
  • employer should be aware of the reaons for collecting personal data on each form
  • employer should be aware of the contents of the privacy policy statement relating to specific collections of recorded information from individuals about themselves

  • DPP6 - Access
